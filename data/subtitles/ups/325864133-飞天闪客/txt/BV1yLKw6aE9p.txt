This is the bedrock of large language model s:
the Transformer architecture;

except that the number of neurons

So, the fix is quite simple,
break it down into many smaller neural networks.
Then, for each new token,
a trained routing layer
directs it to one of those smaller neural networks.
This way, the total parameter count stays the same,

That is the classic MO E Mixture of Experts architecture ,
DeepSeek even put their own spin on it ,

Kimi this time also uses an M O E architecture,

Of course, a classic structure, Residual Connection,
can fix this issue quite well.
So, outside each layer’s transformation,

But there’s a snag: it’s too rigid.
For instance, by the time info from A one reaches A four,
not much is left;
so, ByteDance came up with a new scheme called HC.
Simply put, they copy A one multiple times,
and what gets passed on is not just added on,

This way, the info passed along
stands a better chance of being retained.
But the snag is, if this learning matrix has no constraints,
its repeated products could spiral out of control again.
So, DeepSeek's m H C

Simply put, it keeps these matrices,

thus, an enhancement to H C.

So, what about the computation issues

Back to this Transformer architecture diagram—indeed,
the biggest issue with million token contexts
is that it balloons the computation for the Attention Layer.
since each token in the Attention Layer
must run an attention calculation

So, how do we cut down on this compute?
Simple! One way is to shrink the vector length,
for example, M Q A, G Q A, M L A, and so on.
But these still
fall under dense
attention—meaning, the number of vectors computed
stays the same;
only the vector length is reduced.
Then, of course, there’s another class
that cuts down on the number of vectors
computed—say, by fixing a computation window to select tokens.
or, via some method, sieve out a few Tokens,
not all of them.
Classic methods say, D S A, S W A C S, H C A, and so on
are termed sparse attention,
where the number of vectors grows sparse.
Then, there is the most potent class
that slashes computational complexity
straight from n squared down to n
this approach is called Linear Attention,
which includes these methods here.
So, K D A here stands for Kimi Delta Attention
an in house Linear Attention scheme from Kimi
that is also been used in this K three model.
Of course, this Linear Attention, in some ways,
reverts back to an RNN like method.
So, typically, every few layers,
a Linear Attention is used
and blended with the normal Attention Layer,
hence, it is called Hybrid Attention overall.
